- App.py is basically the backend server, so when it runs it begins a Flask web server on the user's local machine at the specified IP. It wait for requests there, and when a POST request arrives at the /chat endpoint, it wakes up and passes it to Ollama, and then sends the reply back as JSON. 
- The script.js file works when the user clicks the Send button, bringing the sendMessage() method into action. This then gets whatever text is in the input textbox, sends it to Flask using fetch(), and then converts the message to JSON while it waits for a response. Once Flask sends the reply back, the file then unpacks the JSON and sends it to the #output element from the webpage. 
- Index.html is the front end that the user interacts with. So it provides the UI for the user to input text and await a response from the Local LLM. In this way, the separation of concerns between app.py, script.js, and index.html allows for a perfectly functional local LLM that can run effectively on a very compact setup, or if you've got more gas, a very effective local LLM that can perform complex tasks on a local setup.
